HOW TO ACTIVATE EACH FEATURE
=========================================

When DB team shares an object name:
1. Open backend/.env
2. Set the env key value (e.g. ORACLE_SUPPLIER_VIEW=OV_SUPPLIER_YSG)
3. Restart backend: npm run dev
4. The API switches from 503 mock -> live Oracle automatically
5. Test via GET /health - shows oracle.connected=true
6. No code changes needed for view-based features

For SP-based features (create/update/approve):
1. Set env key (e.g. ORACLE_PO_CREATE_SP=SP_PO_CREATE_YSG)
2. Align p_ parameter names with actual SP signature (edit poProcedure.binds.ts if needed)
3. Test via POST /api/purchase-orders with valid body
4. Check backend logs for Oracle errors if 500 returned
